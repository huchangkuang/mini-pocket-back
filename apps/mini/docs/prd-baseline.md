# 简易口袋 PRD 基线文档（V1）

## 1. 产品定位

- 产品名称：简易口袋。
- 产品形态：基于 Taro 的轻工具集合小程序。
- 核心价值：在“低学习成本、即开即用”的前提下，提供高频轻量工具能力，覆盖临时决策、互动娱乐、效率辅助等场景。

## 2. 目标用户

- 个人用户：需要快速完成随机选择、二维码生成、节拍练习等日常小任务。
- 朋友/聚会场景用户：需要快速互动（手持弹幕、指尖轮盘、猜数字）。
- 轻创作用户：需要将图片转为拼豆网格并导出 Excel。

## 3. 核心场景

1. 线下互动：聚会时通过手持弹幕/指尖轮盘快速组织互动。
2. 生活决策：通过“做个决定”在多选项中随机给出结果。
3. 实用工具：快速把文本转二维码并保存到相册。
4. 训练辅助：节拍器辅助乐器练习。
5. 轻创作导出：图片像素化并导出拼豆参考表。

## 5. 模块级输入/输出与限制

### 5.1 手持弹幕

- 输入：文案、字体大小、字体颜色、背景颜色、滚动时间、弹幕类型。
- 输出：全屏展示页渲染结果。
- 限制：文案长度上限 20；依赖路由参数序列化传递。

### 5.2 做个决定

- 输入：标题 + 选项列表，可编辑和保存常用模板。
- 输出：转盘旋转结果（视觉随机）。
- 限制：依赖本地存储维护常用配置；历史项由前端本地管理。

### 5.3 二维码生成

- 输入：最长 500 字符文本。
- 输出：二维码 Canvas 和相册保存。
- 限制：保存依赖相册写权限；异常时仅提示，不做重试队列。

### 5.4 节拍器

- 输入：预设频率或自定义频率（1-208）。
- 输出：音频节拍与节奏高亮反馈。
- 限制：依赖小程序音频上下文；后台/打断场景恢复策略较弱。

### 5.5 拼豆图片生成

- 输入：图片文件、像素粒度参数。
- 输出：颜色网格预览 + 可保存/打开的 Excel 文件。
- 限制：受 Excel 行列上限限制；大图/小像素时性能压力较高。

## 6. 非功能需求（基线）

- 可用性：核心页面应可在主流微信环境稳定打开并完成主流程。
- 性能：单页面首屏交互建议在 2 秒内可用（非严格 SLA）。
- 兼容性：以微信小程序为主，保留 Taro 多端构建能力。
- 可维护性：新增核心能力时必须同步更新 PRD 与技术文档映射。

## 7. 已知问题与迭代建议

- 功能入口一致性：`lottery` 页面注册但入口被隐藏，建议明确产品策略（恢复/下线）。
- 遗留模块治理：`mine` 页面存在未注册，建议明确是否保留。
- 质量保障：页面级自动化测试覆盖不足，建议先为关键流程补冒烟用例。
- 体验细节：部分功能错误提示单一，可补充更明确的引导文案。

## 8. 文档维护机制（强制）

以下变更发生时，必须同步更新本 PRD 文档：

1. 新增页面或新增核心交互能力。
2. 已有能力的目标用户、主流程、关键约束发生变化。
3. 功能入口（分类页、路由、开关策略）发生调整。

维护规则：

- 每次需求迭代在评审阶段确认 PRD 受影响章节。
- 合并前完成“功能映射表”与“已知问题/建议”同步更新。
- 文档修订需附 3 行以内变更摘要，便于回溯。
